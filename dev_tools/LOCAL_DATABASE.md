# 📁 نسخة محلية من قاعدة البيانات - SQLite

## نظرة عامة

تم إنشاء نسخة محلية من قاعدة البيانات باستخدام **SQLite**. هذا يسمح لك بـ:

- ✅ الاختبار بدون الاتصال بالإنترنت
- ✅ العمل بسرعة أكبر (بدون تأخير الشبكة)
- ✅ تجنب استخدام الإنترنت أثناء التطوير
- ✅ حفظ البيانات محليًا

---

## الملفات الجديدة

| الملف | الوصف |
|------|-------|
| `lib/local-db.js` | اتصال قاعدة البيانات المحلية |
| `sync-to-local.js` | نسخ البيانات من Turso إلى SQLite |
| `test-local-db.js` | اختبار قاعدة البيانات المحلية |
| `get-schema.js` | عرض شيما قاعدة البيانات |
| `local_db/users.db` | ملف قاعدة البيانات SQLite |
| `.env.local.example` | مثال على إعدادات البيئة |

---

## خطوات الاستخدام

### 1️⃣ **مزامنة البيانات (خطوة واحدة فقط)**

```bash
node sync-to-local.js
```

هذا سينسخ جميع البيانات من Turso إلى SQLite محليًا:
- 144 مستخدم
- 155 جهة اتصال
- 144 قدرة مستخدم
- 236 متخصصة
- 38 رمز مستخدم

**المجموع:** 717 سجل

### 2️⃣ **تفعيل النسخة المحلية**

أضف أو عدّل في `.env`:

```env
USE_LOCAL_DB=true
```

أو ببساطة:

```bash
export USE_LOCAL_DB=true
```

### 3️⃣ **اختبر النسخة المحلية**

```bash
node test-local-db.js
```

**النتيجة المتوقعة:**
```
✅ جميع الاختبارات نجحت
✅ قاعدة البيانات المحلية تعمل بشكل صحيح
✅ يمكنك الآن استخدام النسخة المحلية للاختبار
```

## استكشاف الأخطاء

### خطأ: "Cannot open database"

```bash
# تأكد من وجود مجلد local_db
mkdir -p local_db

# أعد المزامنة
node sync-to-local.js
```

### خطأ: "better-sqlite3 not installed"

```bash
npm install better-sqlite3
```

### البيانات قديمة

```bash
# احذف الملف القديم
rm local_db/users.db

# أعد المزامنة
node sync-to-local.js
```
